# 功能现状矩阵

最后核对时间：2026-04-08

这份文档只回答一件事：

- 你刚才列的那些能力，当前项目里到底有没有
- 如果有，是“真做完”还是“展示版 / 过渡版”

状态说明：

- `已做`：前后端和数据已成闭环，可以作为当前版本能力对外使用
- `部分做`：已有页面 / 接口 / 表结构 / 演示链路，但还没到正式上线标准
- `未做`：当前项目里没有实际功能闭环

## 5. 当前真正已经有的营销能力

如果只看“已经有基础闭环”的营销功能，目前主要是这两块：

- 优惠券中心：支持领券、我的券、结算页选券、下单用券
- 分销基础版：支持分销中心展示、团队展示、佣金展示、商品分销开关、邀请绑定、下单归因快照、确认收货后累积分佣记录

## 6. 最值得优先补的顺序

如果目标是尽快上线并且真的能用，建议优先级是：

1. 先补“提现申请 / 审核 / 打款状态”
2. 再补“锁客期 / 归因规则后台化”
3. 然后再做会员等级
4. 积分体系放在会员之后
5. 砍价 / 助力 / 抽奖 / 社群裂变最后做

原因很简单：

- 你现在最接近可落地的是分销，不是会员和积分
- 会员、积分、裂变玩法一旦一起上，规则会迅速变复杂
- 先把现有一级分销第一版收稳，再补提现和规则化能力，比一口气堆很多玩法更适合上线

## 7. 结论

一句话总结：

- 分销：`部分做（已有一级归因第一版）`
- 会员：`基本未做`
- 积分：`未做`
- 裂变拉新：`大部分未做`

如果下一步继续推进，最合理的是先把“现有一级直推分佣第一版”继续收稳，再补提现和规则后台化。
