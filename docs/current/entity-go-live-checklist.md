## 二、建议推进顺序

建议按这个顺序推进，不要并行发散：

1. 先确认企业主体基础资料包
2. 注册或确认企业主体下的小程序账号
3. 完成小程序主体校验 / 微信认证
4. 提交小程序备案
5. 完成腾讯云企业实名认证
6. 购买腾讯云 MySQL、开通微信云托管或云资源
7. 把正式 `WECHAT_APP_ID / WECHAT_APP_SECRET`、数据库连接串补到生产环境
8. 联调真实微信登录
9. 申请并接入微信支付
10. 做正式环境回归、提审、发布

- 企业对公银行账户信息

- 公司公章
- 若负责人不是法人，准备授权书签章能力

如果后面要做支付，还建议一起准备：

- 财务联系人
- 客服电话
- 售后地址

## 四、小程序账号与微信认证需要的资料

### 1. 注册小程序时

企业主体注册小程序时，官方文档说明企业类型账号可选两种主体验证方式：

- 对公账户打款验证
- 微信认证验证，需支付 300 元认证费

当前项目更适合企业主体直接做正式账号，不建议继续沿用演示 `AppID`。

你们要准备的资料：

- 未注册过公众平台的邮箱
- 企业营业执照
- 企业主体信息
- 小程序管理员信息
- 若走对公打款，准备企业对公账户并在 10 天内完成打款
- 若走微信认证，准备微信支付扫码付款

### 2. 微信认证时

根据微信官方“小程序认证指引”，企业类型通常至少要准备：

- 《营业执照》
- 若未三证合一，可能还需要《组织机构代码证》
- 对公账户信息
- 发票信息

常见注意点：

- 微信认证费用除政府及部分免费组织外，一般都要付审核服务费
- 审核通常为 1 到 5 个工作日
- 审核期间要保持认证联系人电话畅通
- 认证资料提交后 30 天内有 3 次补充 / 重填机会
- 认证有效期 1 年，后续要年审

如果公司名称刚变更：

- 需要工商变更证明

## 五、小程序备案需要的资料

### 1. 必备资料

按微信官方“小程序备案操作指引”和“备案材料示例”，你们至少要准备：

- 主办单位证件
- 主体负责人证件
- 小程序负责人证件
- 小程序负责人现场人脸核身
- 前置审批文件（若业务类目涉及）
- 补充材料：授权书、承诺书、情况说明等

对于企业主体，最常见就是：

- 营业执照原件彩色扫描件 / 拍照件
- 法人身份证正反面
- 小程序负责人身份证正反面
- 主体负责人手机号
- 小程序负责人手机号
- 主办单位详细地址
- 小程序名称、服务内容、类目说明

### 2. 备案资料格式要求

微信官方当前要求里，比较容易卡审的是这些：

- 证件需为原件彩色扫描件或原件拍照
- 不能用复印件、黑白件、翻拍屏幕件、电子证件截图
- 证件有效期通常需大于 3 个月
- 授权书 / 承诺书签署日期应与提交备案日期一致，或小于 1 个月

### 3. 授权书相关

备案最容易被忽略的是“负责人是否必须是法人”这个省份差异：

- 某些省份主体负责人必须是法定代表人
- 某些省份允许授权，但必须上传主体负责人授权书
- 某些省份小程序负责人可不是法人，但要上传小程序负责人授权书
- 上海地区小程序负责人授权书由平台自动生成，其他地区常常要自行下载模板并盖章签字

所以备案省份一旦确定，最好按该省规则单独再核一遍授权材料。

### 4. 时间节奏

微信官方当前公开流程是：

1. 备案信息填写
2. 平台初审，通常 1 到 2 个工作日
3. 工信部短信核验
4. 通管局审核，通常 1 到 20 个工作日
5. 备案成功后才能进入下一步版本发布等流程

短信核验重点：

- 短信来自 `12381`
- 收到后 24 小时内必须完成核验
- 最大核验次数 5 次
- 超时或失败会被退回，需要重新提交

## 六、腾讯云企业实名认证资料

如果你们按当前项目路线走“微信云托管 + 腾讯云 MySQL”，那不一定要先买 CVM，但一定要先完成腾讯云企业实名认证。

腾讯云官方当前企业实名认证方式主要有 4 种：

- 企业法人微信扫码认证：即时完成
- 企业法人人脸识别认证：即时完成
- 腾讯云充值认证：约 1 个工作日
- 企业对公打款认证：约 1 到 5 个工作日

### 1. 法人微信扫码 / 法人人脸识别

这两种方式准备资料最轻，通常要：

- 企业营业执照彩色件
- 企业名称
- 统一社会信用代码 / 组织机构代码
- 行业信息
- 通讯地址
- 企业联系人姓名
- 企业联系人身份证号
- 企业联系人手机号
- 最新法定代表人信息

额外要求：

- 微信扫码方式需要法人本人微信操作
- 扫码授权时，提交实名信息要与当前微信支付实名预留信息一致
- 官方当前提示暂不支持中国港澳台及国际法人

### 2. 对公打款认证

如果不用法人微信扫码 / 人脸识别，就要额外准备：

- 企业对公银行账户
- 开户银行
- 开户支行
- 银行开户地址
- 法定代表人姓名
- 法定代表人身份证号
- 法人手持身份证国徽面彩色照
- 法人手持身份证人像面彩色照
- 经办人信息
- 经办人人脸识别

流程上要注意：

- 腾讯云会向企业对公账户打 0.01 到 0.99 元随机金额
- 需要在发起认证后 7 个自然日内回填金额
- 连续两次金额填错会失败

### 3. 和 ICP 的关系

腾讯云官方明确提醒：

- 腾讯云实名认证名称必须与后续备案主体名称一致，否则无法申请 ICP 备案

## 七、买服务器 / 域名时还要不要额外准备资料

### 1. 如果继续走当前项目路线：微信云托管

你们其实不一定要买传统云服务器。

当前仓库已经明确偏向：

- 微信云托管部署后端
- 腾讯云 MySQL 做数据库

这条路线下，前期只要：

- 腾讯云企业实名认证
- 开通数据库
- 开通云托管
- 配好环境变量

如果前端继续通过 `wx.cloud.callContainer` 调后端，前期可以先不绑定自定义公网域名。

### 2. 如果要买 CVM / 轻量服务器

买腾讯云服务器本身，核心还是先完成上面的企业实名认证。资料不会比云托管少，通常也不会额外多出一套企业材料。

但如果后面要：

- 绑定自定义域名
- 做官网 / H5
- 让公网域名承载生产流量

那就不是“企业认证”了，而是会进入 **域名实名认证 + ICP 备案** 的链路。

### 3. 自定义域名 / ICP 备案要额外准备的东西

如果你们后续要把商城、官网或 API 绑到自定义域名上，通常还要额外准备：

- 域名实名信息
- 域名证书 / 域名持有证明
- 备案主体资料
- 网站 / 服务负责人资料
- 接入商要求的核验材料
- HTTPS 证书

而且微信云托管官方说明里写得比较明确：

- 自定义域名用于生产访问时，需要先完成域名备案，否则无法访问

## 八、和当前项目直接相关、马上要补的正式信息

结合当前仓库配置，主体到位后需要尽快落地到项目里的信息有：

- 正式小程序 `AppID`
- 正式小程序 `AppSecret`
- 生产 `DATABASE_URL`
- 最终使用的腾讯云账号 / 云环境归属
- 是否继续沿用当前云托管环境 ID 和服务名

当前仓库里已经能看到但还不算最终生产落地的信息：

- 小程序当前 `project.config.json` 里已有一个 `AppID`
- 小程序当前配置已切到 `requestTransport: "cloud"`
- 小程序当前配置里已有 `cloud.env` 和 `cloud.service`
- 服务端 `WECHAT_APP_ID / WECHAT_APP_SECRET` 仍是空位

所以主体信息到位后，第一件技术动作不是继续补页面，而是把正式环境参数收口。

## 九、项目视角下的结论

你们现在真正还差的，不是商城代码主体，而是这几件现实前置：

1. 企业主体资料包收齐
2. 小程序正式账号和微信认证
3. 小程序备案
4. 腾讯云企业实名认证
5. 正式数据库和云托管资源
6. 真实微信登录参数
7. 微信支付

如果只追“最快可上线”：

- 小程序备案是硬前置
- 微信支付是最后一段硬前置
- 自定义域名 ICP 可以后置，前提是你们前期不走自定义域名
