# 商城产品蓝图与 V1 范围

更新时间：2026-03-31

这份文档合并了原来的产品蓝图和增长范围说明，目标是只回答 3 件事：

1. 这个项目要做成什么
2. 第一版到底收在哪里
3. 哪些东西明确放到二期以后

## 一句话定位

当前项目最适合定义为：

- `微信小程序商城 + 优惠券 + 一级分销 + 基础复购运营`

第一版的核心不是“玩法很多”，而是：

- 能成交
- 能履约
- 能登录
- 能支付
- 能沉淀基础增长数据

## 首发总原则

如果目标是尽快上线并且真的能卖货，第一版建议坚持这 5 条：

1. 主链路优先于玩法。
2. 真实履约优先于展示页面。
3. 一级分销优先于复杂裂变。
4. 优惠券优先于积分体系。
5. 直播和跨端明确不进首发关键路径。

## 首发必做范围

### 商城基础

- 首页、分类、搜索、商品详情
- 购物车、结算、下单、订单、订单详情
- 地址管理
- 售后申请

### 基础运营

- 领券中心
- 我的优惠券
- 结算选券
- 订单取消退券

### 上线基础设施

- 真实微信登录
- 腾讯云 MySQL
- 微信云托管
- 微信支付
- 最小履约后台

## 增长系统的 V1 收口

| 模块 | 第一版建议保留 | 先不要做太重 | 当前状态 |
| --- | --- | --- | --- |
| 拉新 | 新人券包、首单激励、来源归因、邀请绑定 | 多层任务体系、复杂老带新奖励 | 部分做 |
| 分佣 | 一级直推、邀请归因、佣金账单、团队沉淀 | 二级分佣、复杂等级、排行榜 | 部分做 |
| 裂变 | 海报分享、商品分享入口、直属成员沉淀 | 砍价、助力、抽奖、群裂变 | 部分做 |
| 积分 | 如果排期宽松可做账户和明细 | 积分商城、积分换实物、复杂任务中心 | 未做 |

这里要明确一条边界：

- 如果首发目标是“最快上线”，积分不建议进入当前主线

原因：

- 当前积分是从零开始
- 它不会阻塞商城首发
- 一旦进入主线，账户、明细、过期、抵扣规则都会明显增复杂度

## 第一版明确不进关键路径的内容

这些能力有价值，但不适合现在强上：

- 二级分佣
- 提现自动打款
- 排行榜
- 复杂会员等级
- 完整积分体系
- 积分商城
- 砍价 / 助力 / 抽奖
- 直播带货
- H5 / 跨端

## 分期建议

### Phase 1：成交上线版

目标：

- 商城主链路稳定
- 登录、支付、履约、售后能闭环
- 优惠券和一级分销具备基础可用性

### Phase 2：增长补强版

目标：

- 分佣结算和提现
- 更真实的分享入口
- 更清晰的邀请沉淀和团队统计
- 基础会员或积分能力

### Phase 3：内容与运营版

目标：

- 更重的营销玩法
- 直播承接
- 更完整的后台运营系统

## 适用品类提醒

这套商城底座更适合：

- 珠宝：高客单、重展示、重信任
- 食品 / 保健品：高复购、重券包、重会员运营

它们可以共用一套底座，但运营重点不同：

- 珠宝更重内容和成交信任
- 食品 / 保健品更重复购和活动运营

## 当前最实用的产品建议

如果只保留几句真正能指导决策的话，就是下面这些：

1. 第一版先把“成交和履约”做稳，不要先堆玩法。
2. 分佣先收成一级直推，不要一上来就做二级。
3. 裂变先做“可追踪的分享”，不要先做活动工厂。
4. 积分不是当前阻塞项，排期紧时直接放二期。
5. 直播明确放二期，但类目判断可以提前准备。
